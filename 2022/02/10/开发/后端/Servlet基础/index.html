Web文件夹结构

Web应用程序基本由以下项目组成

  • 静态资源(HTML、图片、视频)
  • Servlet
  • JSP
  • 自定义类
  • 工具类
  • 部署描述文件(web.xml)、设置信息(Annotation等)

web应用程序有一特殊文件夹/WEB-INF ,其中的资源不可直接请求，如起中的web.xml部署描述文件 classes 文件夹 存放了自定义类 lib文件夹放置应用到的JAR文件。

/META-INF/resources文件夹中可以放置静态资源或JSP等，如果First项目收到某次请求First/index.html,但实际上项目中不存在First下的index.html，则会使用resources中的index.html。(tips:好像仅能用于index.html的请求)

如果要用到某个类，Web应用程序先在/WEB-INF/classes中尝试加载类,若无，试从/WEB-INF/lib的Jar文件中搜索类文件，若无，容器实现本身存放的类或JAR的文件中搜索。

getParameterMap();

import java.io.FileNotFoundException;
import java.io.FileOutputStream;
import java.io.IOException;
import java.io.InputStream;
import java.io.OutputStream;
import java.util.regex.Matcher;
import java.util.regex.Pattern;
import javax.servlet.ServletException;
import javax.servlet.annotation.MultipartConfig;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.Part;

@MultipartConfig //必须设置此标注才能使用getPart()相关API，否则getPart()返回null
@WebServlet(
		name="uploadphoto",
		urlPatterns = "/photo.php.py.java"
		)
public class Photo extends HttpServlet{
	private static final Pattern fileNameRegex=Pattern.compile("filename=\"(.*)\"");
	@Override
	protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
		resp.getWriter().print("error");
	}
	@Override
	protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
		Part photo = req.getPart("photo");
		String filename = req.getParameter("filename");
		if(filename==null||filename.equals("")) {
			filename = getSubmittedFileName(photo);
		}
		boolean is = write(photo, filename);
		if(is) {
			resp.getWriter().print("D:/"+filename);
		}else {
			resp.getWriter().print("error in writer");
		}
	}
    //通过标头获取文件的名称
	private String  getSubmittedFileName(Part part) {
		String header = part.getHeader("Content-Disposition");
		System.out.println(header);
		Matcher matcher = fileNameRegex.matcher(header);
		matcher.find();
		String filename = matcher.group(1);
		if(filename.contains("\\")) {
			return filename.substring(filename.lastIndexOf("\\"));
		}
		return filename;
	}
    //写入文件，默认到D盘
    /*
    *虽然在Servlet3.1中Part新增了getSubmittedFileName(),可以直接取得上传文件名，但是由于问浏览器上传文件的差异性，上传的文件名可能含有不安全的字符。
    */
	private boolean write(Part photo,String filename) throws IOException,FileNotFoundException {
		try(InputStream in = photo.getInputStream();
				OutputStream out = new FileOutputStream("D:/"+filename);){
			byte[] buffer = new byte[1024];
			int length = -1;
			while((length=in.read(buffer))!=-1) {
				out.write(buffer,0,length);
			}
			return true;
		}catch (Exception e) {
			return false;
		}
	}
}

@MultipartConfig标注可以用来设置Servlet处理上传文件的相关信息，可用属性有:

fileSizeThreshold:整数值设置，默认为0，上传文件超出设置，会先写入缓存。

location: 字符串设置，设置写入目录。

maxFileSize:限制单个文件的大小，默认值为-1L,表示不限制(单位：字节)。

maxRequestSize:限制multipart/form-data请求的(表单)大小，默认-1L，表示不限制(单位：字节)。

tips:在设置文件表单的限制大小后，当提交的文件或表单大小超限Servlet就会抛出异常。

如果要通过web.xml设置@MultipartConfig的属性

<servlet>
    <servlet-name>name</servlet-name>
    <servlet-class>cn.name</servlet-class>
    <multipart-config>
        <location>c:/worspace</location>
    </multipart-config>
</servlet>

protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
	req.setCharacterEncoding("UTF-8");
	resp.setContentType("text/html; charset=UTF-8");
	req.getParts()
		.stream()
		.filter(part->part.getName().startsWith("photo"))

protected void write(Part part){
	String filename =part.getSubmittedFileName();
	filename = filename.substring(filename.lastIndexOf("."));//获取后缀名
	try {
		part.write(String.format("%s%s",Instant.now().toEpochMilli(),filename));//文件名为时间+后缀名
	} catch (Exception e) {
		// TODO: handle exception
	}
}

会话管理

使用Cookie

Cookie是servlet发送到Web浏览器的少量信息，由浏览器保存，然后发送回服务器。cookie的值可以唯一标识客户端，因此cookie通常用于会话管理。cookie有名称、单个值和可选属性，如注释、路径和域限定符、最大使用年限和版本号。

一些Web浏览器在处理可选属性方面存在bug，所以要尽量少使用它们，以提高servlet的互操作性。servlet通过使用HttpServletResponse.addCookie (javax.servlet.http.Cookie)方法发送Cookie，它在HTTP响应头中添加字段，以便向浏览器发送Cookie，每次一个。该浏览器预计将为每个Web服务器支持20个cookie，总计300个cookie，并可能将cookie大小限制在每个4kb。浏览器通过在HTTP请求头中添加字段将cookie返回给servlet。可以通过使用Ht tpServletRequest从请求中检索cookie。

getCookies()方法。多个cookie可能具有相同的名称但不同的路径属性。cookie会影响使用它们的Web页面的缓存。HTTP 1.0不缓存使用这个类创建的cookie的页面。这个类不支持HTTP 1.1中定义的缓存控件。

返回值 方法 描述
Object clone() 重写标准方法以返回此 Cookie 的副本。
String getComment() 返回描述此 Cookie 用途的注释，或者如果该 Cookie 没有注释返回null

protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

例如

登录：

@WebServlet("/login")
public class LoginSession extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.setContentType("text/html; charset=UTF-8");
        PrintWriter out = resp.getWriter();
        HttpSession session = req.getSession(true);
        session.setMaxInactiveInterval(300);
        String name = req.getParameter("name");
        if(null==name){
            name="游客";
        }
        session.setAttribute("name",name);
        out.println("Welcome,"+name);
    }
}

登出：

@WebServlet("/logout")
public class LogoutSession extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        HttpSession session = req.getSession(true);
        String name = (String)session.getAttribute("name");
        resp.setContentType("text/html; charset=UTF-8");
        if(name==null){

            resp.getWriter().println("no login");
            return;
        }
        resp.getWriter().println(name+"成功登出");
        session.invalidate();
    }
}

在Servlet中直接使用


@WebServlet(name="ServletConfigTest", urlPatterns = "/conf",
        initParams = {
            //设置param1和param2的值
            @WebInitParam(name="param1",value="value1"),
            @WebInitParam(name="param2",value = "value2")
        }
)
public class ServletConfigTest extends HttpServlet {
    private  String param1,param2,param3;
    @Override
    //通过重写init方法获取param1和param2
    public void init() throws ServletException{
        //getServletConfig()返回ServletConfig对象
        param1 = getServletConfig().getInitParameter("param1");
        param2 = getServletConfig().getInitParameter("param2");
    }
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        resp.getWriter().println("param1:"+param1+"param2:"+param2+"param3"+param3);//输出测试
    }
}

在web.xml中配置

<servlet>
    <!--名称与Servlet中设置的名称一样,才可以覆盖设置-->
    <servlet-name>ServletConfigTest</servlet-name>
    <servlet-class>test.ServletConfigTest</servlet-class>
    <init-param>
        <!--name一样，覆盖在@WebServlet中的设置-->
        <param-name>param1</param-name>
        <param-value>value1_1</param-value>
    </init-param>
    <!--不设置param2的值，保留原本的值
    <init-param>
        <param-name>param2</param-name>
        <param-value>value2_2</param-value>
    </init-param>-->
    <init-param>
        <!--name不一样，新设置param3的值为value3_3-->
        <param-name>param3</param-name>
        <param-value>value3_3</param-value>
    </init-param>
</servlet>
<servlet-mapping>
    <!--覆盖url-->
    <servlet-name>ServletConfigTest</servlet-name>
    <url-pattern>/configs</url-pattern>
</servlet-mapping>

ServletContextListener

ServletContext接口有两个默认方法

public interface ServletContextListener extends java.util.EventListener {
    default void contextInitialized(javax.servlet.ServletContextEvent sce) { /* compiled code */ }
    default void contextDestroyed(javax.servlet.ServletContextEvent sce) { /* compiled code */ }
}

Web应用程序在初始化后获即将结束销毁前会调用实现了该接口的类的这两个方法，所以可以在contextInitialized中初始化资源,在 contextDestroyed中释放资源。

如:

实现ServletContextListener接口

@WebListener
public class ServletContextTest implements ServletContextListener {
    @Override
    public void contextInitialized(ServletContextEvent sce) {
        //取得ServletContext
        ServletContext context = sce.getServletContext();
        //取得web.xml设置的初始参数
        String avatar = context.getInitParameter("avatar");
        //设置ServletContext属性
        context.setAttribute("avatar",avatar);
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {

    }
}

web.xml配置

<!--Serlet3.0前需要配置此项
<listener>
    <listener-class>test.Listener.ServletContextTest</listener-class>
</listener>
-->
<context-param>
    <param-name>avatar</param-name>
    <param-value>avatar_value</param-value>
</context-param>

public interface ServletContextAttributeListener extends java.util.EventListener {

@WebListener()
public class ServletContextAttributeListenerTest implements ServletContextAttributeListener {

        System.out.println("add");
    }

    @Override

    }
