Git 学习

Git仓库得创建,提交

创建版本库

git init

添加文件道暂存区

将工作区的内容提交到暂存区

git add filename

提交到本地仓库

将暂存区的内容提交到本地仓库

git commit -m "information"

版本间得时光穿梭

查看当前仓库状态

git status

查看具体的修改

git diff

版本退回

查看提交日志

git log

回退版本

一个^表示向历史退一个版本，多个^表示退回多个版本

git reset --hard Head^

当退回版本过多,如100

git reset --hard~100

或者通过 commit id ”时光穿梭“ 到某个版本

Tips：commit id 不必写全，写一部分即可

git reset --hard 12345

可以通过以下命令查看命令记录从而得到commit id

git reflog

丢弃工作区修改

一种是filename自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是filename已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

git checkout -- filename

当当前工作区有文件被删除时，可以使用以下命令删除仓库中的相应文件

rm filename

远程仓库

添加远程仓库

创建SSH Key

tips:一路回车就好

ssh-keygen -t rsa -C "email@example.com"

Windows用户就可以在C:\Users\用户名\.ssh目录下找到相关的id_rsa.pub (公匙),和id_rsa (私匙)。

在远程仓库添加公匙

登录GitHub、Gitee等，在相关repo中添加公匙即可

关联远程仓库

link 可以是远程仓库的https(如：https://gitee.com/User/test.git)，也可以是SSH(如: git@gitee.com:User/test.git)

git remote add origin link

推送内容

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

git push -u origin master

添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。

把本地库的内容推送到远程，用git push命令，实际上是把当前分支master推送到远程。

从现在起，只要本地作了提交，就可以通过命令：

git push origin master

删除远程库

如果推送错误，可以通过以下方式删除远程库

1.查看远程库信息

git remote -v

2.根据名字删除本地仓库与远程仓库的绑定关系

git remote rm origin

仓库克隆

git clone link

分支管理

创建分支

方法一 ：

创建并切换到dev分支

git branch dev #创建分支
git checkout dev #切换分支

也可以只使用这条命令

git checkout -b dev

方法二：

还可以使用switch创建切换分支，更科学

创建并切换到dev分支：

git switch -c dev

切换到分支dev：

git switch dev

查看所有分支

git branch

分支合并

将指定分支合并到当前分支

git merge dev

删除分支

git branch -d dev

解决冲突

当两个分支提交了不同的内容到仓库，然后进行合并时，就会出现：

Automatic merge failed; fix conflicts and then commit the result.
自动合并失败;修复冲突，然后提交结果。

此时打开冲突的文件，手动修改后重新提交，就能合并成功。

使用以下命令就可以查看分支合并情况。

git log --graph --pretty=oneline --abbrev-commit

分支管理策略

fast forward合看不出来曾经做过合并,在分支合并时git在可行的情况下会使用fast forward ,删除分支后，会丢掉分支信息。可以强制禁用fast forward ，git就会在merge时生成新的commit，这样分支历史上就可以看出。

git merge --no-ff -m "merge with no-ff" dev
